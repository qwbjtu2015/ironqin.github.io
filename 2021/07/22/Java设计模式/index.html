综述设计模式

设计模式是程序设计模式的一种总结和最佳实践。按照类型可以分为创建型、结构型和行为型。还有一种类型Sun Java Center确定的J2EE模式，关注于表示层。

顾名思义，创建型就是

单例模式

单例模式（Singleton Pattern）是Java中最简单的设计模式之一。这种类型的设计模式属于创建型模式，它提供了一种创建对象的最佳方式。

这种模式涉及到一个单一的类，该类负责创建自己的对象，同时确保只有单个对象被创建。这个类提供了一种访问其唯一的对象的方式，可以直接访问，不需要实例化该类的对象。

  • 构造器是private修饰
  • 提供一个静态方法获取它的唯一的实例
  • 一般使用双重锁校验进行修饰

工厂模式

定义一个创建对象的接口，让其子类决定实例化哪一个工厂类，工厂模式使其创建过程延迟到子类进行。

使用条件：我们明确地计划不同条件下创建不同实例时。

使用场景：日志记录器；数据库访问；连接服务器的框架，需要三个协议POP3 IMAP HTTP，可以将这三个作为产品类，共同实现一个接口。
