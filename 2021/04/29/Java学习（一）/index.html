Java学习(一)

Posted on 2021-04-29 |
Java基础知识复习要点

1 基础部分

  1. Java动态性：反射机制
  2. Java三个版本：JavaSE（桌面程序、控制台）、JavaME（Android）、JavaEE（web开发、服务器开发）
  3. JDK（开发工具）包含JRE（运行环境），JRE包含JVM（虚拟机）
  4. 编译型（C、C++、Java）和解释型：（JS、Python）
  5. 运行机制：java源代码（.java）编译成为字节码(.class)，JVM（加入类装载器，然后字节码校验、最后解释成机器语言交给操作系统执行）
  6. Java是强类型语言
  7. 八种基本数据类型
    • byte 1个字节
    • short 2个字节
    • int 4个字节
    • long 8个字节
    • float 8个字节
    • double 8个字节
    • char 2个字节
    • bool 1个位
  8. 引用类型: 类 接口 数组
  9. 整数扩展：int也可以表示二进制(0b)、八进制(0)、十进制、16进制(0x)
  10. 浮点数扩展：绝对避免浮点数直接比较，应使用BigDecimal类
  11. Java中的字符实际都是数字
  12. 类型转换：强制类型转换（类型（变量名）->从高转低）、自动类型转换（由低变高）
  13. 不能对bool进行转换；不能转换为不相干类型；高到低强制，低到高自动。
  14. 数字之间可以用下划线分割，大数计算溢出问题
  15. 类的实例变量如果没有初始化，会输出默认值。Int默认0，布尔值默认false
  16. JavaDoc @author @version @since @param @return @throws
  17. 在编译时指定文件编码Javadoc -encoding UTF-8 -charset UTF-8 Doc.java
  18. Java输入函数Scanner中的next方法以空格作为一次读取的分隔符，而且字符串前面和后面的空格无法读取，nextLine是以回车符作为分隔
  19. 形参与实参：方法中的形式参数和实际参数
  20. 方法重载：在同一个类中，相同函数名，不同函数参数（包括参数类型或数量或排列顺序）
  21. 命令行传参数：main函数里面的args中使用
  22. 可变参数：一个方法中只能有一个可变参数，且只能在最后一个参数，如 int… a double… a,本质就是数组，也可直接传递数组
  23. 递归：注意递归深度，太深导致栈溢出，应当避免
  24. Java内存：堆：存放new出来的对象，栈存的是函数、变量引用
  25. 面向对象编程OOP：以类的方式组织代码，以对象的组织封装数据
  26. 三大特性：封装、继承、多态（接收同样的信息或者调用相同的方法，但是表现出（输出）不同的状态）
  27. 类的构造方法：方法名称与类名完全一致，且没有返回值。功能是实例化对象。 每个类默认有一个构造方法（无参构造），如果定义了有参构造，那么默认的构造器将会失效。不同的构造器属于重载
  28. 类的成员变量即使不初始化，也会有默认值，一般是0，u0000, false, null
  29. set和get方法：
    • 提高程序安全性，包括有效性验证等
    • 隐藏实现细节
    • private变量属性私有（防止用户直接操作变量）
  30. 继承：单继承：一个类只能有一个父类，子类继承父类，即获得了父类的所有public方法和public变量。所有的类都默认直接或间接继承自Object类
  31. super：super.father_attr 可以使用super关键字+点+public或protected型的属性和方法
  32. 子类实例化时会首先调用父类构造方法，然后调用子类构造方法，如果没有显式写出，则默认调用父类的无参构造。如果父类没有无参构造函数，则子类也无法使用无参构造，如果子类使用无参构造，但没有显式写明使用父类哪个构造方法，则会默认使用父类的无参构造。super(age)
  33. 方法重写Override：子类重写父类方法，方法名、返回值类型和参数列表必须完全一致，而且必须是非静态方法，父类的private方法不可能被重写，即使子类有同名方法，也不叫重写，修饰符的范围可以扩大，但不能缩小。比如父类的方法是protected，子类的重写方法可以是public，但不能是private。重写原因：父类的功能，子类不一定需要或者说不一定一致。
  34. 多态： 父类的引用指向子类的对象，b可以调用的方法是要B类有的，但是如果A复写了某方法，那么b会调用子类复写后的方法，如果子类没有复写该方法，则会执行父类的方法，因为子类继承了所有的父类方法。如果是子类和父类都有的static同名方法，b调用的则是B类的方法。
  35. 多态：同一方法可以根据发送对象的不同而采取多种不同的行为方式。一个对象的实际类型是确定的，但可以只想对象的引用的类型可以有很多。（该对象所属类的所有父类）
  36. 多态存在的条件：有继承关系；子类重写父类的方法；父类引用指向子类的对象
  37. 类型转换：子类转父类需要强制转换，因为有可能丢失方法；父类转换为子类可以直接转换
  38. static关键字：静态变量：属于类的变量 静态方法：可以直接用类名直接调用，不用实例化再调用。静态代码块：随类加载时执行，且只执行一次。匿名代码块就是不加static的代码块，这个是在每个类实例创建时都会执行的代码，可以用来初始化实例信息
  39. 抽象类：abstract既可以修饰方法（抽象方法） 也可以修饰类（抽象类）。抽象类中可以没有抽象方法，但是有抽象方法的类必须为抽象类。抽象类不能被实例化，但是可以被继承，他的子类可以是抽象类，也可以不是抽象类，如果不是抽象类，他就必须实现父类的所有抽象方法。即使一个抽象类里面没有抽象方法，他也不能直接被实例化。
  40. 抽象类是有构造方法的，这是因为子类继承他时肯定要执行父类的构造方法
  41. 接口：抽象类中还有部分非抽象方法，而接口中全部都是抽象方法，也就是只有编程规范，所以接口中的方法默认都是抽象方法
  42. 接口中的变量默认都是public static final修饰，但一般不在接口中定义变量。接口不是类，所以肯定没有构造方法。接口可以多实现，类只能单继承
  43. 内部类：成员内部类、静态内部类、局部内部类、匿名内部类
  44. 异常：Throwable是所有异常的超类，有Error和Exception两个子类

    Error由Java虚拟机抛出，异常分为运行时异常和非运行时异常，运行时异常需要我们在程序中捕获
  45. 捕获和抛出异常：try,catch,finally,throw,throws; try和catch代码块是必写的，finally是可选的，一般是善后工作，比如关闭打开的文件，释放资源等等。无论是有没有异常都会执行finally中的代码。Throw是制造一个异常，throws是抛出一个异常。抛出异常的作用是在编辑器中提醒调用该方法的调用者处理这个异常，调用者可以处理，也可以继续向上抛出，但是main函数没必要再网上抛出了，不然就跑到了虚拟机中

2 集合类与IO

  1. Java集合类 只能存放对象的引用，如果是基本类型，则要存对应的封装类（int->Integer），包含Set、List和Map三类。Set是无序不重复，List是有序可重复，Map是映射关系
  2. HashSet 不是线程安全的，集合元素可以是null。Hasset是将该对象的HashCode进行hash映射，决定存放位置。可以存不同类型的数据
  3. TreeSet 有序集合，排序时调用元素的compareTo方法比较元素大小关系，TreeSet需要元素是相同类型。如果TreeSet存的是某个类，这个类需要重写compareTo方法，或者new一个Comparator接口的实现类的对象：
  4. Collection接口->List接口->ArrayList类， ArrayList’也是线程不安全
  5. HashMap和HashTable，HashTable较为古老，且不允许null作为key，HashMap线程不安全。TreeMap TreeMap的初始化必须传入一个Comparator对象
  6. Collections集合工具类：sort排序，reverse翻转，replaceAll替换所有A元素替换全部B元素
  7. String StringBuffer StringBuilder:三个字符串相关的类，均为final类型，不可被继承。String定义的字符串不可被改变，StringBuffer是线程安全的，StringBuilder是线程不安全，因此操作短字符串使用String类，单线程下操作大量字符使用StringBuilder，多线程下操作大量字符使用StringBuffer
  8. File类：文件、文件名以及路径的抽象表示，可以使用相关方法对文件、文件夹进行操作（创建、删除、判断、遍历、文件大小）
  9. IO：Input：输入（读取）Output：输出（写入） 流：字符（字节）1个字符=2个字节。从内存->硬盘：输入，从硬盘->内存：输出。根据流中数据的类型分为字节流和字符流。
    ||输入流|输出流|
    |字节流|字节输入流InputStream|字节输出流OutputStream|
    |字符流|左幅输入流Reader|字符输出流Writer|
    字节流：一切皆为字节，二进制保存，字节流可以读取任意文件
    计算机上的所有文本编辑器（记事本、notepad、sublime等）在打开文件时都会去查询编码表，将字节转换为字符进行显示，如果字节范围均在0-127，则会查询ASCII表，否则会查询系统的默认编码表，比如中文的GBK。
    String转为字节数组：“hello”.getBytes()
    UTF-8中文：一个汉字3个字节，GBK中文：一个汉字2个字节
    默认模式是覆盖写文件，append模式是追加写
    Windows换行符是\r\n，Linux换行符是\n，Mac换行符是\r
    Java的IO操作首先提交给JVM，再由JVM提交给OS，OS调用自己的IO操作方法进行操作。
    FileInputStream和FileOutputStream类：读取写入的是byte或byte数组。
    文件读取，每次读取1024个字节
  10. 字符流：Java封装了对字符的读取器Reader，每次读取一个字符，无论是中文字符还是英文字符。当然也可以读取多个字符。FileWriter和FileReader读取和写入的是char或char数组。FileWriter的write方法是将字符写入了内存，并没有直接写入文件，因此可以手动调用flush将内存中的字符转成字节写入文件。同时close方法会先调用flush再关闭，所以如果既没有调用flush有没有调用close，那么写入数据将会失败
  11. 缓冲流BufferedInputStream和BufferedOutputStream比原来的FileInputStream和FileOutputStream速度更快，建立了缓冲区
  12. 字符编码和字符集：字符编码：二进制与字符之间的转换关系（对应规则）。字符集（charset）：一个系统所支持的所有字符的集合，包括各国文字、标点、符号等。ASCII字符集对应ASCII编码表，GBK字符集对应GBK编码，Unicode字符集对应UTF-8/UTF-16/UTF-32编码表。ASCII只有127个字符，ISO-8859-1使用单字节编码。GB2312只有简体字，GBK支持简体繁体中文汉字以及日韩汉字。Unicode称为万国码，国际统一编码，UTF-8中的每个字符占据3个字节
  13. 转换流：以指定的字符编码读取或写入文件（InputStreamReader和OutputStreamWriter）
  14. 序列化流和反序列化流，类似于Python的Pickle包，写入或读取的是对象，也叫对象的序列化，这个都是字节。这个对象的类应该implements Serializable接口。该接口没有抽象方法，因此无需实现方法。在读取序列化对象时，应该有该对象对应的类的class文件，否则抛出ClassNotFound异常
  15. Transient：瞬态关键字，static是静态关键字，static对应的成员变量是无法被序列化的。而transient修饰的变量同样也是不能被序列化的，但是不具有static的含义，只是标记不被序列化而已
  16. InvalidClassException：反序列化时读取到的对象的类结构如果发生了变化，那么就会抛出InvalidClassException。Java对两个类结构判断的通过serialVersionUID来判断，如果我们想忽略类的修改造成的该异常，可以在类定义中添加该变量，必须是static和final修饰的long类型
  17. 打印流：System.out.println()其中的out就是打印流PrintStream的对象，PrintStream继承自OutputStream字节输出流。实现了print、println、printArray等方法，同时永远不会抛出IOException，可以打印任意类型的值
