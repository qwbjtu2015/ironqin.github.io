Java学习(五)-注解与反射

Posted on 2021-05-15 |
Java注解与反射

  1. 注解（Annotation）定义：

    • 它不是程序本身，可以对程序作出解释（给人看）
    • 可以被其他程序读取（如编译器）（给程序看）
    • 格式：“@注释名”，还可以添加参数值如@SuppressWarnings(value="unchecked")
    • 使用场景：可以附加在package，class，method和field上面，相当于为他们添加了额外的辅助信息，我们可以通过反射机制编程实现对这些元数据的访问。

  2. 内置注解

    • @Override 重写方法的注解，只能修饰方法
    • @Deprecated 废弃方法的注解，不推荐使用，但可以使用。可以修饰方法、属性、类。
    • @SuppressWarnings 用来抑制编译时的警告信息，这个注解需要添加参数才可用

  3. 元注解：负责解释其他注解，Java定义了4个标准的元注解，用来对其他注解类型做说明

    • @Target 描述注解的适用范围
    • @Retention 表示需要在什么级别保存该注解信息，用于描述注解的声明周期【Source<Class<Runtime】
    • @Documented 说明该注解将被包含在javadoc中
    • @Inherited 说明子类可以继承父类中的该注解

  4. 自定义注解
    使用@interface自定义注解，会自动继承Annotation接口

    • 其中每个方法实际上时声明了一个配置参数，方法名称就是参数名称，类似于方法，实际上不是
    • 返回值类型就是参数的类型（返回值只能是基本类型，Class,String，enum）
    • 可以通过default声明参数的默认值
    • 如果只有一个参数，一般参数名为value
    • 可以通过反射读取注解

  5. 反射机制Reflection

    • 动态语言：在运行时可以改变其结构的语言，运行时才可以确定其数据类型，如PHP、Asp、JavaScript、Python、Perl
    • 静态语言：运行时结构不可变的语言是静态语言（C、C++、Java），数据类型在编译时已经确定。Java不是动态语言，但可以算“准动态语言”，因为Java可以利用反射机制获得类似动态语言的特性。
    • Reflection（反射）允许程序在执行期间借助于反射API获取任何类的内部信息，并能直接操作任意对象的内部属性及方法，包括其私有的属性和方法。（因此使得安全性受到一定影响）
    • 类被加载之后，在堆内存的方法区中就产生了一个Class类型的对象（一个类只有一个Class对象），这个对象就包含了完整的类的结构信息。
    • 正常方式：导入包名下的类->通过new实例化对象->取得实例化对象
    • 反射方式：一个实例化对象->getClass()方法->得到完整的类名称和信息

  6. 反射提供的功能

    • 在运行时判断任意一个对象所属的类
    • 在运行时构造任意一个类的对象
    • 在运行时判断任意一个类所具有的全部变量和方法
    • 在运行时获取泛型信息
    • 在运行时调用任意一个对象的变量和方法
    • 在运行时处理注解
    • 生成动态代理（面向切面编程AOP）
    • 优点：实现动态创建对象和编译，灵活性增强
    • 缺点：对性能有影响，比new方式的对象创建慢了几十倍

  7. 反射相关的主要API

    • java.lang.Class 代表一个类
    • java.lang.reflect.Method 代表类的方法
    • java.lang.reflect.Field 代表类的变量
    • java.lang.reflect.Constructor 代表类的构造器
    • …

  8. Class类：

    • Class本身是一个类
    • Class对象只能由系统建立，我们通过多种方式获取该对象
    • 一个加载的类在JVM中只会有一个Class实例
    • 一个Class对象对应的是一个加载到JVM中的.class文件
    • 每个类的对象都会记录自己是有哪个Class实例生成的
    • 通过Class可以完整地得到一个类中的所有被加载的结构
    • Class类是反射的根源，只有获得对应的Class对象才能在运行时改变类的行为

    • class 类、局部内部类、匿名内部类、数组、枚举、基本数据类型
    • interface 接口 注解
    • void

  12. 类加载的内存分析

    • 类的加载过程：加载->链接->初始化
    • 加载：将class文件字节码加载到内存中，将静态数据转换成方法区的运行时数据结构，然后生成该类的Class对象
    • 链接：将Java类的二进制代码合并到HVM的运行状态之中
      • 验证：确保加载的类信息符合JVM规范
      • 准备：正式为类变量（static）分配内存并设置默认初始值
      • 解析：
    • 初始化
      • JVM执行类构造器的clinit()方法，自动收集类变量的赋值动作和静态代码块中的语句合并产生
      • 初始化类时，如果其父类还没有进行初始化，则要先触发其父类的初始化
      • 虚拟机会保证一个类的clinit方法在多线程环境中被正确加锁和同步

  13. 类的初始化发生的场景

    • 类的主动引用（会发生初始化）
      • 当虚拟机启动时，先初始化包含main方法的类
      • 当new一个类的对象时
      • 当调用类的静态成员和静态方法时（final常量除外！！）
      • 使用java.lang.reflect包的方法对类进行反射调用时
      • 当初始化一个类时，如果其父类没有初始化，则会触发父类的初始化
    • 类的被动引用（不会发生初始化）
      • 当子类引用父类的静态变量时不会触发子类的初始化
      • 通过数组定义类的引用时，不会触发初始化
      • 引用常量时不会触发初始化（final）

  14. 类加载器的作用：将类装载进内存

    • 类加载的作用：将class文件加载到内存中，并将这些静态数据转换成方法区的运行时数据结构，然后在堆中生成一个代表该类的java.lang.Class对象，作为方法区中类数据的访问入口。
    • 类缓存：标准的JavaSE类加载器可以按要求查找类，某个类一旦被加载到类加载器中，它将维持加载一段时间，JVM垃圾回收机制可以回收这些Class对象。
    • 源文件（.java）经过编译成为字节码文件（\.class），而后经过类加载器和字节码校验器以及解释器最后交给操作系统执行。

  15. JVM中定义的几种类加载器

    • 引导类加载器(Bootstap ClassLoader)：JVM自带的类加载器，负责Java核心库，rt.jar
    • 扩展类加载器(Extension ClassLoader): lib/ext目录下的jar包
    • 系统类加载器(System ClassLoader):常用的类加载器

  16. 双亲委派机制：为了安全起见，系统会一层一层验证，尽量寻找较高层级类加载器中包含的类，首先从下往上，从系统加载器->扩展类加载器->根加载器，然后再从根加载器->扩展类加载器->系统加载器，寻找最高层级的类加载器。比如用户定义了java.lang.String类，系统不会使用用户自定义的String类，而是根加载器中的String类。

双亲委派加载的顺序：
- 类加载器收到类加载的请求
- 当前加载器将该请求向上委托给父类加载器，一直委托到根加载器（Bootstrap）
- 根加载器检查是否能加载这个类，能的话直接加载，否则通知子加载器进行加载
- 向上委托，向下加载，过程类似于递归
17. 获取运行时对象的信息（Field属性、Method方法、Constructor构造器、Superclass父类、Interface接口、Annotation注解）
- getName() 包名+类名
- getSimpleName() 类名
- getFields() 所有的public属性
- getDeclaredFields 全部属性，包括private
- getMethods() 获取该类和它所有父类的public方法
- getDeclaredMethods() 获取该类所有的方法,包括private
- getMethod(“setName”, String.class) 获取指定的方法
- getField() 获取指定的变量
- getConstructors() 获取public的构造方法
- getDeclaredConstructors() 获取所有构造器
18. 动态创建对象执行方法

  1. 性能对比分析
    • Method、Field、Constructor对象都有setAccessible()方法
    • setAccessible作用是是否进行可用性检查
    • 我们可以将该方法设置为true，来避免每次的安全检查
    • 经过测试，直接通过对象调用：反射不关闭检查调用：反射关闭检查调用=3:1700:1200
  2. 反射操作泛型
    • Java采用泛型擦除的机制来引入泛型，Java中的泛型仅仅是给编译器使用的，一旦编译完成，所有和泛型相关的类型全部擦除
    • 为了通过反射操作这些类型，Java新增了ParameterizedType/GenericArrayType/TypeVariable和WildcardType几种类型来代表不能被归一到Class类但又和原始类型齐名的类型
    • ParameterizedType：一种参数化类型，如List
    • GenericArrayType:一种元素类型是参数化类型或类型变量的数组类型
    • TypeVariable：各种类型变量的公共父接口
    • WildcardType：一种通配符类型表达式
